## 引言
最开始接触CTF Crypto的时候，我看着“RSA”“AES”这些名词，再加上一堆数学公式，直接有点劝退——总觉得这玩意儿又难又抽象，离实战太远。但真正沉下心学下来发现，Crypto其实是“入门易、精通难”，核心是先把基础框架搭起来，再通过真题慢慢打磨。

这篇文章是我整理的Crypto前置学习笔记，不是纯理论堆砌，而是结合了自己踩过的坑、常用的工具和真题场景——不管你是刚入门的新手，还是想补基础的同学，看完都能直接上手解题，为后续学RSA共模攻击、AES模式漏洞这些进阶内容铺路。

## 一、先搞懂：学Crypto要达到什么目标？
其实不用一开始就追求“精通所有算法”，入门阶段核心是搞定3件事：
1. 能快速判断CTF题目是哪种加密类型（古典密码/对称/非对称/哈希）；
2. 掌握必备的数学工具（模运算、GCD、逆元），知道怎么用代码实现；
3. 熟练用3个以上工具，能独立破解古典密码和基础对称/非对称加密题。

顺便说下CTF Crypto的题型分布，心里有个数：
- 古典密码（15%左右）：凯撒、Vigenère这些，入门必练，难度最低；
- 基础对称加密（25%左右）：主要是AES的各种模式漏洞，比如ECB、CBC；
- 基础非对称加密（30%左右）：RSA占大头，常考共模攻击、低指数攻击；
- 哈希/签名（20%左右）：无盐密码破解、哈希碰撞这些，工具就能搞定；
- 现代密码协议（10%左右）：高阶内容，入门阶段不用急着学。

## 二、密码学核心概念：用大白话讲明白
密码学本质就是“在不安全的地方安全传信息”，几个核心术语不用死记，结合场景理解就行：
- 明文：没加密的原始信息，CTF里基本都是`flag{xxx}`这种格式，比如`flag{caesar_cracked}`；
- 密文：加密后的数据，可能是纯字母（古典密码）、十六进制串（AES/RSA），或者Base64编码后的样子；
- 密钥：加密/解密的“钥匙”，可能是数字（凯撒的偏移量）、字符串（Vigenère的关键词），也可能是16进制串（AES的128位密钥）；
- 算法：加密的规则，CTF里不用深究原理，记“特征”就行——比如AES密文长度一定是16字节的整数倍；
- 密码分析：就是我们解题的过程——拿到密文，根据算法特征找漏洞，还原出明文或密钥。

还有两个原则要记住，对解题很有帮助：
1. Kerckhoffs原则：算法是公开的，安全全靠密钥——所以CTF题目里，算法一定是已知的（要么题目说，要么能通过密文猜），重点是找密钥；
2. 安全强度：比如56位的DES已经能暴力破解了，现在主流是128位AES，CTF里遇到短密钥的题目，优先试试爆破。

## 三、必备数学基础：不用啃公式，会用就行
Crypto（尤其是RSA）离不开数学，但真不用深究推导过程，重点是掌握“怎么算”“怎么用代码实现”，下面这几个是高频考点：

### 3.1 素数与因数分解
素数就是只能被1和自己整除的数（比如2、3、7），RSA的安全就靠“大整数难分解”——但CTF里的n大多是小整数，比如n=1234567，直接分解成127×9721就行。

我常用的分解工具是sympy库，一行代码搞定：
```python
from sympy import factorint
print(factorint(1234567))  # 输出：{127:1, 9721:1}

    3.2 模运算：简化大计算的核心

    模运算就是求余数，记为a mod n，比如123 mod 17=4（因为17×7=119，123-119=4）。

    关键是记住两个运算律，CTF里能省很多事：

    • (a×b) mod n = [(a mod n)×(b mod n)] mod n
    • (a+b) mod n = [(a mod n)+(b mod n)] mod n

    比如算(123×456) mod 17，不用硬算123×456=56088，直接算(4×14) mod17=56 mod17=5，结果一样。

    3.3 GCD与扩展欧几里得算法

    GCD是最大公约数，比如gcd(12,18)=6，gcd(17,5)=1（这种叫互质）。RSA里要求e和φ(n)互质，就是靠GCD判断的。

    扩展欧几里得算法更实用，能找到x、y使得a×x + b×y = gcd(a,b)，主要用来求“模逆元”——RSA解密的核心步骤。

    刚开始学的时候，我回代步骤总算错，后来总结了个简单的代码模板，直接复用：

    def extended_gcd(a, b):
    if b == 0:
        return (a, 1, 0)
    g, x1, y1 = extended_gcd(b, a % b)
    x = y1
    y = x1 - (a // b) * y1
    return (g, x, y)

# 示例：求17和5的GCD及x、y
g, x, y = extended_gcd(17, 5)
print(f"gcd={g}, x={x}, y={y}")  # 输出：gcd=1, x=-2, y=7

    3.4 模逆元：RSA解密必备

    如果a×x ≡ 1 mod n，那x就是a在模n下的逆元，比如3×5=15≡1 mod7，所以3在模7下的逆元是5。

    求逆元不用手动算，sympy的mod_inverse函数直接出结果：

    1
2
3

    from sympy import mod_inverse
inv = mod_inverse(3, 7)
print(inv)  # 输出：5

    这里踩过个坑：只有a和n互质时才有逆元，要是不互质，mod_inverse会报错，这时候就要考虑其他攻击方法了。

    3.5 快速幂：计算a^b mod n的高效方法

    如果直接算2^100 mod 7，肯定会溢出，快速幂能把复杂度降到O(logb)，Python里甚至有内置函数pow(a, b, n)，直接用就行：

    1

    print(pow(2, 100, 7))  # 输出：2，不用算2^100的超大数

    四、三大核心加密技术：分清场景，对症解题

    4.1 对称加密：一把钥匙用到底

    对称加密就是加密和解密用同一个密钥，速度快，适合加密大文件，但密钥要靠安全信道传——这也是它的痛点。

    CTF里最常考的是AES，尤其是ECB模式，因为它有个致命漏洞：相同的明文块加密后，密文块也相同，能直接泄露明文结构。

    比如用AES-ECB加密重复的明文“hellohellohello”，密文会出现重复的16字节块，一眼就能看出用了ECB模式：

    1
2
3
4
5
6
7
8

    from Crypto.Cipher import AES
from Crypto.Util.Padding import pad

key = b"1234567890abcdef"  # 16字节AES-128密钥
plaintext = b"hellohellohello"  # 重复明文块
cipher = AES.new(key, AES.MODE_ECB)
ciphertext = cipher.encrypt(pad(plaintext, 16))
print(ciphertext.hex())  # 输出会有重复的16字节串

    另外，DES因为密钥只有56位，现在已经能暴力破解了，CTF里遇到DES题目，直接用John the Ripper工具爆破就行。

    4.2 非对称加密：公钥私钥配对用

    非对称加密有两把钥匙：公钥（公开给所有人）和私钥（自己藏好）。公钥加密的内容，只有私钥能解密；私钥签名的内容，只有公钥能验证。

    CTF里主要考RSA，安全基础是“大整数难分解”。常见的漏洞场景有两种：

    1. 低指数攻击（e=3）：如果明文m比较小，m³ ≡ c mod n，直接开立方就能得到m；
    2. 共模攻击：同一明文用不同公钥加密，拿到两个密文就能还原明文（后面会专门写真题解析）。

    举个低指数攻击的例子：已知n=1234567，e=3，c=12345，直接开立方得m=23，验证一下pow(23, 3, 1234567)确实等于12345，搞定。

    4.3 哈希函数：数据的“数字指纹”

    哈希函数能把任意长度的数据，转换成固定长度的哈希值——比如MD5输出32位十六进制串，SHA256输出64位。

    它的核心特点是“单向性”：不能从哈希值反推明文；还有“抗碰撞性”：不同明文很难得到相同的哈希值。

    CTF里常考无盐密码破解——比如网站把密码直接MD5哈希存储，没有加“盐”（随机字符串），我们就能查彩虹表（预存的哈希-明文对）破解。

    比如已知哈希值5f4dcc3b5aa765d61d8327deb882cf99，查彩虹表直接知道明文是“password”，验证一下：

    1
2

    import hashlib
print(hashlib.md5(b"password").hexdigest())  # 输出就是上面的哈希值

    这里提醒下：MD5和SHA1已经被破解了，现在安全场景都用SHA256，但CTF里老题还会遇到MD5。

    五、古典密码：入门必练，找解题感觉

    古典密码是CTF Crypto的“开胃菜”，安全性不高，但能帮你培养“找特征、用工具”的解题思维，下面这3种最常考：

    5.1 凯撒密码：暴力破解就行

    原理很简单：字母按固定偏移替换，比如偏移3位，A→D、B→E。

    破解的时候根本不用算，CyberChef直接搞定：

    1. 打开CyberChef（https://gchq.github.io/CyberChef/）；
    2. 拖入“ROT13”模块，把“Rotate by”改成1-25的范围；
    3. 输入密文，一下子就能看到所有可能的明文，找flag{}格式的就行。

    比如密文“khoor zruog”，偏移3位就是“hello world”。

    5.2 Vigenère密码：多表替换，先找关键词长度

    比凯撒密码复杂点，用关键词控制偏移——比如关键词“key”，就是k→10、e→4、y→24，循环加密明文。

    破解步骤：

    1. 先找重复的密文块，算间隔的最大公约数，判断关键词长度（比如间隔是6，关键词可能是2、3、6位）；
    2. 按关键词长度分组，每组按凯撒密码破解；
    3. 工具直接用dcode.fr（https://www.dcode.fr/vigenere-cipher），选“Auto-decrypt”，自动识别关键词和解密。

    我第一次做Vigenère题的时候，手动算关键词长度算错了，后来发现直接用工具自动识别，效率高多了。

    5.3 栅栏密码：按行数排列再读取

    原理是把明文按“栅栏行数”排成几行，再按列读取。比如明文“flag{rail}”，行数2，排列就是： f l g { r a i l 按列读就是“fglrail{”，这就是密文。

    破解的时候暴力尝试行数（通常2-5行），CyberChef的“Rail Fence Cipher”模块直接调，换几个行数就出来了。

    六、CTF Crypto必备工具：少而精，够用就好

    不用装一堆工具，这几个就够覆盖80%的题目：

    6.1 在线工具（零基础首选）

    1. CyberChef：万能工具，古典密码、编码转换、哈希计算都能搞定，界面直观，拖模块就行；
    2. dcode.fr：专门破解古典密码，Vigenère、Hill密码这些，自动识别能力强；
    3. HashKiller：破解无盐哈希，MD5、SHA1都能查，输入哈希值直接搜明文。

    6.2 本地工具（进阶必备）

    1. Python库：
      • pycryptodome：实现AES/RSA，安装命令pip install pycryptodome，注意用的时候要给明文padding，不然会报错；
      • sympy：数学计算（因数分解、逆元），安装pip install sympy；
      • pwntools：CTF专用库，快速幂、模运算都封装好了，安装pip install pwntools；
    2. OpenSSL：命令行工具，AES/RSA加密解密，比如AES-ECB解密： 
      1

      openssl enc -aes-128-ecb -d -in ciphertext.bin -out plaintext.txt -K 1234567890abcdef1234567890abcdef

    七、解题通用思路：从真题里总结的经验

    分享一个我常用的解题流程，不管遇到什么题，按这个步骤来，能少走很多弯路：

    1. 先识别题型：看密文格式（纯字母→古典密码，十六进制→AES/RSA，32位串→MD5）；
    2. 尝试基础破解：古典密码用CyberChef暴力试，哈希值查彩虹表；
    3. 找漏洞：对称加密看模式（ECB优先看重复块），RSA看密钥长度（短密钥→分解，e=3→低指数）；
    4. 验证结果：破解后一定要验证明文是不是flag{}格式，或者用加密算法反向验证（比如RSA解密后，用公钥加密看看是不是原来的密文）。

    举个例子，攻防世界的新手题“凯撒密码”：

    • 密文“khoor”，纯字母→判断凯撒；
    • CyberChef调ROT模块，偏移3位→明文“hello”；
    • 题目完整密文是“khoor zruog”，解密后是“hello world”，最终flag是flag{hello_world}。

    八、学习路径与资源推荐：少走弯路

    8.1 我的学习顺序（由浅入深）

    1. 先刷5道古典密码题（攻防世界新手区），熟悉工具使用；
    2. 学对称加密，重点练AES-ECB/CBC模式的漏洞题；
    3. 啃RSA基础，先做低指数、共模攻击这些简单题；
    4. 最后学哈希和签名，补充高阶内容。

    8.2 好用的学习资源

    • 视频：B站“CTF Crypto入门教程（从0到1）”，里面真题演示很直观，比看书好懂；
    • 博客：先知社区Crypto板块，很多大佬会分享解题思路；
    • 书籍：《CTF Crypto实战指南》，不搞虚的，全是实战技巧；
    • 刷题平台：攻防世界（新手）、PicoCTF（基础）、CryptoCTF（高阶）。

    最后说两句

    Crypto入门不难，难的是坚持练题和总结。刚开始可能会遇到“看不懂题目”“工具用不顺”的情况，不用急，每道题都记下来自己踩过的坑，慢慢就有感觉了。

    这篇前置基础学好后，后面会陆续更新RSA共模攻击、AES-CBC模式漏洞、哈希碰撞这些进阶内容，跟着练下来，Crypto题型基本能拿下大半。如果有哪里没看懂，或者遇到具体题目卡壳了，也可以留言交流~
